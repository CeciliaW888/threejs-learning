Core Concepts

The Full Pipeline

User types "cozy reading nook" → API generates JSON → Three.js creates scene. This is the core of our AI portfolio generator. Today we connect all the pieces.

Key Components

The pipeline has four distinct components, each with a clear responsibility:

1
API Client

Calls Claude with structured prompts. Handles authentication, sends the user's text description along with a system prompt that enforces JSON output. The client manages rate limiting and retries so the rest of the pipeline doesn't have to worry about API reliability.

2
JSON Parser

Validates and extracts scene data from the API response. This is your safety net — it catches malformed JSON, missing required fields, and unexpected data shapes before they crash your Three.js code. Always validate at this boundary.

3
Scene Builder

Creates Three.js objects from the validated JSON. Maps geometry types (box, sphere, cylinder) to Three.js classes, applies materials and positions, sets up lights, and configures the camera. This is the bridge between data and visuals.

4
UI Controller

Manages user interaction, loading states, and error display. Shows a spinner while the API call is in progress, displays error messages if something goes wrong, and disables the generate button to prevent duplicate requests.

How Async Calls Work Without Blocking the UI

API calls take 1-5 seconds. If you block the main thread during that time, the 3D canvas freezes and the UI becomes unresponsive. The solution is async/await — JavaScript's way of saying "start this task, but keep the animation loop running while we wait."

The pipeline flow looks like this:

  1. User clicks "Generate" → UI shows loading state
  2. await callAPI(prompt) → browser continues rendering while waiting
  3. API returns JSON → parser validates the data
  4. Scene builder clears old objects and creates new ones
  5. UI hides loading state → user sees the new scene
Key Insight

The async/await pattern is essential here. The animation loop runs on requestAnimationFrame (separate from the main thread's microtask queue), so your 3D scene keeps rendering smoothly even while the API call is in progress.

Clearing vs. Updating Scenes

When a new scene is generated, you need to remove the old one first. Simply adding new objects without clearing creates a mess of overlapping geometry. The clearScene() method removes all children from the scene, including lights — so you must re-add everything from the new scene data.

Key Terminology

Text-to-3D Pipeline
The complete flow from user text input through API call, JSON parsing, to rendered 3D scene. Each stage transforms data into a different format.
API Client
A wrapper class that manages communication with an external API. Handles authentication, request formatting, and error recovery.
JSON Validation
Checking that parsed JSON matches an expected schema (required fields, correct types) before using it. Prevents runtime crashes from malformed AI output.
Scene Builder
A class that converts JSON scene descriptions into Three.js objects (meshes, lights, camera positions). The bridge between data and visuals.
Loading State
Visual feedback shown to the user while an async operation (like an API call) is in progress. Prevents confusion and duplicate submissions.
Async/Await
JavaScript syntax for handling asynchronous operations. Lets you write code that "waits" for results without blocking the main thread or freezing the UI.
Scene Clearing
Removing all objects, lights, and helpers from a Three.js scene before building a new one. Essential to avoid overlapping geometry from previous generations.

Code Examples

Complete Pipeline: AISceneGenerator Class

AISceneGenerator.js Complete Example 
import * as THREE from 'three';
import Anthropic from '@anthropic-ai/sdk';

class AISceneGenerator {
  constructor(apiKey) {
    this.client = new Anthropic({ apiKey });
    this.scene = new THREE.Scene();
    this.camera = new THREE.PerspectiveCamera(
      75, window.innerWidth / window.innerHeight, 0.1, 1000
    );
    this.renderer = new THREE.WebGLRenderer();
    this.renderer.setSize(window.innerWidth, window.innerHeight);
    document.body.appendChild(this.renderer.domElement);
  }

  async generateFromText(prompt) {
    try {
      // 1. Show loading state
      this.showLoading();

      // 2. Call API
      const sceneData = await this.callAPI(prompt);

      // 3. Clear existing scene
      this.clearScene();

      // 4. Build new scene from JSON
      this.buildScene(sceneData);

      // 5. Render
      this.render();

      this.hideLoading();
      return { success: true };

    } catch (error) {
      this.hideLoading();
      this.showError(error.message);
      return { success: false, error: error.message };
    }
  }

  async callAPI(prompt) {
    const message = await this.client.messages.create({
      model: 'claude-3-5-sonnet-20241022',
      max_tokens: 1024,
      system: `Generate 3D scene JSON. Schema:
{
  "objects": [{"type": "box"|"sphere"|"cylinder", "position": [x,y,z], "color": "#hex", "scale": [x,y,z]}],
  "lights": [{"type": "ambient"|"directional"|"point", "position": [x,y,z], "intensity": number, "color": "#hex"}],
  "camera": {"position": [x,y,z], "lookAt": [x,y,z]}
}`,
      messages: [{ role: 'user', content: prompt }]
    });

    return JSON.parse(message.content[0].text);
  }

  buildScene(data) {
    // Add objects
    data.objects.forEach(obj => {
      let geometry;
      switch(obj.type) {
        case 'box': geometry = new THREE.BoxGeometry(...obj.scale); break;
        case 'sphere': geometry = new THREE.SphereGeometry(obj.scale[0]); break;
        case 'cylinder': geometry = new THREE.CylinderGeometry(...obj.scale); break;
      }

      const material = new THREE.MeshStandardMaterial({ color: obj.color });
      const mesh = new THREE.Mesh(geometry, material);
      mesh.position.set(...obj.position);
      this.scene.add(mesh);
    });

    // Add lights
    data.lights.forEach(light => {
      let lightObj;
      switch(light.type) {
        case 'ambient':
          lightObj = new THREE.AmbientLight(light.color, light.intensity);
          break;
        case 'directional':
          lightObj = new THREE.DirectionalLight(light.color, light.intensity);
          lightObj.position.set(...light.position);
          break;
        case 'point':
          lightObj = new THREE.PointLight(light.color, light.intensity);
          lightObj.position.set(...light.position);
          break;
      }
      this.scene.add(lightObj);
    });

    // Position camera
    this.camera.position.set(...data.camera.position);
    this.camera.lookAt(new THREE.Vector3(...data.camera.lookAt));
  }

  clearScene() {
    while(this.scene.children.length > 0) {
      this.scene.remove(this.scene.children[0]);
    }
  }

  render() {
    this.renderer.render(this.scene, this.camera);
  }

  showLoading() {
    document.getElementById('loading').style.display = 'block';
  }

  hideLoading() {
    document.getElementById('loading').style.display = 'none';
  }

  showError(message) {
    const errorEl = document.getElementById('error');
    errorEl.textContent = 'Error: ' + message;
    errorEl.style.display = 'block';
  }
}

// Usage
const generator = new AISceneGenerator(process.env.ANTHROPIC_API_KEY);
document.getElementById('generateBtn').onclick = () => {
  const prompt = document.getElementById('prompt').value;
  generator.generateFromText(prompt);
};
What's happening: The AISceneGenerator class encapsulates the entire pipeline. The generateFromText method orchestrates the flow: show loading → call API → clear scene → build scene → render → hide loading. Each method has a single responsibility.

Adding an Animation Loop

Animation loop addition 
// Add this method to AISceneGenerator
animate() {
  requestAnimationFrame(() => this.animate());

  // Rotate all meshes slowly for visual interest
  this.scene.children.forEach(child => {
    if (child.isMesh) {
      child.rotation.y += 0.01;
    }
  });

  this.renderer.render(this.scene, this.camera);
}

What's happening: After generating a scene, this animation loop keeps objects rotating slowly. It only affects meshes (not lights), creating a gentle showcase effect. The arrow function preserves the this context.

Hands-On Exercises

Exercise 1: Build the Full Pipeline

~45 min

Goal: Create a working text-to-3D system from scratch

  1. Set up an HTML page with an input field, generate button, loading indicator, and error display
  2. Implement the AISceneGenerator class from the code example above
  3. Test with different prompts: "a forest clearing", "a modern kitchen", "floating geometric shapes"
  4. Add error handling for invalid JSON — wrap JSON.parse in a try/catch and show the user a helpful message
  5. Verify that the loading state appears and disappears correctly

Expected Output: Type a description, click generate, see a loading indicator, then watch a 3D scene appear in the canvas.

Exercise 2: Add the Animation Loop

~15 min

Goal: Make generated scenes more dynamic with continuous animation

  1. Add the animate() method to your AISceneGenerator class
  2. Call generator.animate() after creating the instance
  3. Experiment: try different rotation speeds, add bobbing (sine wave on Y position), or have objects scale up/down
  4. Make sure the animation doesn't interfere with scene generation — new scenes should replace the old animated ones

Exercise 3: Add JSON Validation and Fallbacks

~20 min

Goal: Make the pipeline robust against malformed API responses

  1. Create a validateSceneData(data) function that checks for required fields: objects (array), lights (array), camera (object with position and lookAt)
  2. If validation fails, fall back to a default scene (a single cube with ambient light)
  3. Log the validation error so you can debug what the API actually returned
  4. Test by temporarily breaking the system prompt to see how your fallback handles bad data
Validation hint 
function validateSceneData(data) {
  if (!data || typeof data !== 'object') return false;
  if (!Array.isArray(data.objects)) return false;
  if (!Array.isArray(data.lights)) return false;
  if (!data.camera || !data.camera.position) return false;
  return true;
}
