Core Concepts

Streaming vs Single Response

Yesterday's pipeline used single response mode: you send a prompt, wait for the complete JSON response, then render everything at once. This works, but the user stares at a loading spinner for 2-5 seconds with no feedback.

Streaming changes the experience: tokens arrive as they're generated, like watching someone type in real-time. For scene modifications, this means you can start applying changes before the full response is ready.

When to use each approach:

  • Single response — Best for generating complete new scenes. You need the full JSON before you can parse and render it.
  • Streaming — Best for modifications and feedback. You can show the AI's "thinking" as text while waiting for the final JSON, giving users confidence that something is happening.

With streaming, the API sends chunks of text as they're generated. You accumulate these chunks in a buffer and attempt to parse the JSON once you detect it's complete (e.g., you've received a closing brace that balances all opening braces).

Modification Command Types

Scene modifications fall into three categories, each requiring a different approach:

Add
Additive Commands

"Add a table," "put a lamp in the corner," "add some trees." These commands create new objects without changing existing ones. The AI needs to know the current scene state so it can position new objects appropriately (not overlapping existing ones). Implementation: parse the new object JSON and call scene.add().

Transform
Transformative Commands

"Make it darker," "rotate the chair," "change the floor to wood," "make everything blue." These modify properties of existing objects. The AI returns a list of changes referencing objects by name. Implementation: find the object in the scene by name and update its properties (position, rotation, material color, light intensity, etc.).

Remove
Subtractive Commands

"Remove the lamp," "delete the table," "clear all the rocks." These remove objects from the scene. The AI identifies which objects to remove by name. Implementation: find the object and call scene.remove(). Don't forget to dispose of geometry and materials to free memory.

Key Insight

The secret to modification commands is scene state. You must pass the current scene's object list to the AI so it knows what exists. Without context, the AI can't know which "chair" to rotate or whether there's already a lamp in the corner.

Undo/Redo: Managing Scene History

Users will make mistakes or want to experiment. Undo/redo is essential for a good creative tool. The simplest approach: store snapshots of the scene state as JSON before each modification.

How it works:

  • Before every modification, push the current scene state (as JSON) onto an undoStack
  • When the user clicks "Undo," pop the last state from undoStack, push the current state onto redoStack, then rebuild the scene from the popped state
  • When the user clicks "Redo," pop from redoStack, push current state onto undoStack, then rebuild
  • Any new modification clears the redoStack (you can't redo after making a new change)

This is known as the Command Pattern — each action is recorded so it can be reversed. For large scenes, you might store diffs instead of full snapshots to save memory.

Key Terminology

Streaming Response
Receiving API output token-by-token as it's generated, rather than waiting for the complete response. Enables real-time feedback and progressive rendering.
Additive Modification
A scene change that creates new objects without altering existing ones. Example: "add a bookshelf" creates a new mesh and adds it to the scene.
Transformative Modification
A scene change that alters properties of existing objects. Example: "make it darker" reduces light intensity, "rotate the chair" changes an object's rotation.
Subtractive Modification
A scene change that removes existing objects. Example: "remove the lamp" finds and deletes the lamp mesh from the scene, including geometry and material disposal.
Scene State
A JSON representation of all objects, lights, and camera settings in the current scene. Passed to the AI so it knows what exists and can make contextual modifications.
Command Pattern
A design pattern where each action is recorded as an object, enabling undo/redo. Each command stores enough information to reverse itself.
Undo Stack
An array of previous scene states. Each modification pushes the current state before changing it. Undo pops the last state and rebuilds the scene from it.

Example 1: Streaming Modification Handler

streaming-modifier.js Core Pattern 
async modifyScene(command) {
  // Pass current scene state so the AI knows what exists
  const stream = await this.client.messages.stream({
    model: 'claude-3-5-sonnet-20241022',
    max_tokens: 512,
    system: `You are modifying an existing 3D scene.
Current scene state: ${JSON.stringify(this.sceneState)}

Output ONLY a JSON object describing the changes:
{
  "action": "add" | "modify" | "remove",
  "targets": [
    {
      "name": "object-name",
      "changes": { "position": [x,y,z], "color": "#hex", ... }
    }
  ],
  "newObjects": [
    { "name": "new-obj", "type": "box", "position": [x,y,z], "color": "#hex", "scale": [x,y,z] }
  ]
}
No explanations. Only JSON.`,
    messages: [{ role: 'user', content: command }]
  });

  let buffer = '';
  for await (const chunk of stream) {
    if (chunk.type === 'content_block_delta') {
      buffer += chunk.delta.text;
      // Try to parse once we have enough data
      this.tryParseAndApply(buffer);
    }
  }
}

tryParseAndApply(buffer) {
  try {
    const changes = JSON.parse(buffer);
    this.applyChanges(changes);
  } catch (e) {
    // Not yet complete JSON, keep accumulating
  }
}
What's happening: The streaming handler receives tokens incrementally. It accumulates them in a buffer and attempts to parse the JSON after each chunk. Once the JSON is complete (and parseable), it applies the changes to the scene. Failed parse attempts are silently caught — they just mean the response isn't complete yet.

Example 2: Applying Modifications by Type

apply-changes.js 
applyChanges(changes) {
  // Save current state for undo
  this.undoStack.push(JSON.stringify(this.sceneState));
  this.redoStack = []; // Clear redo on new change

  switch (changes.action) {
    case 'add':
      // Create and add new objects
      changes.newObjects.forEach(obj => {
        const mesh = this.createMesh(obj);
        mesh.name = obj.name;
        this.scene.add(mesh);
        this.sceneState.objects.push(obj);
      });
      break;

    case 'modify':
      // Update existing objects
      changes.targets.forEach(target => {
        const object = this.scene.getObjectByName(target.name);
        if (!object) return;

        if (target.changes.position) {
          object.position.set(...target.changes.position);
        }
        if (target.changes.color) {
          object.material.color.set(target.changes.color);
        }
        if (target.changes.rotation) {
          object.rotation.set(...target.changes.rotation);
        }
        if (target.changes.scale) {
          object.scale.set(...target.changes.scale);
        }

        // Update scene state to match
        const stateObj = this.sceneState.objects.find(o => o.name === target.name);
        if (stateObj) Object.assign(stateObj, target.changes);
      });
      break;

    case 'remove':
      // Remove objects from scene
      changes.targets.forEach(target => {
        const object = this.scene.getObjectByName(target.name);
        if (!object) return;

        // Dispose of geometry and material to free memory
        if (object.geometry) object.geometry.dispose();
        if (object.material) object.material.dispose();
        this.scene.remove(object);

        // Remove from state
        this.sceneState.objects = this.sceneState.objects.filter(
          o => o.name !== target.name
        );
      });
      break;
  }
}
What's happening: This method handles all three modification types. For additions, it creates new meshes and adds them. For transforms, it finds objects by name and updates their properties. For removals, it properly disposes of Three.js resources before removing. Each change also updates the sceneState so the AI always has an accurate picture.

Example 3: Undo/Redo Implementation

undo-redo.js 
class SceneHistory {
  constructor(sceneBuilder) {
    this.builder = sceneBuilder;
    this.undoStack = [];
    this.redoStack = [];
    this.maxHistory = 20; // Limit memory usage
  }

  // Call this before every modification
  saveState(currentState) {
    this.undoStack.push(JSON.stringify(currentState));

    // Trim if too many states stored
    if (this.undoStack.length > this.maxHistory) {
      this.undoStack.shift();
    }

    // New action clears redo history
    this.redoStack = [];
  }

  undo(currentState) {
    if (this.undoStack.length === 0) return null;

    // Save current state for redo
    this.redoStack.push(JSON.stringify(currentState));

    // Restore previous state
    const previousState = JSON.parse(this.undoStack.pop());
    this.builder.rebuildFromState(previousState);
    return previousState;
  }

  redo(currentState) {
    if (this.redoStack.length === 0) return null;

    // Save current state for undo
    this.undoStack.push(JSON.stringify(currentState));

    // Restore next state
    const nextState = JSON.parse(this.redoStack.pop());
    this.builder.rebuildFromState(nextState);
    return nextState;
  }

  canUndo() { return this.undoStack.length > 0; }
  canRedo() { return this.redoStack.length > 0; }
}

// Wire up keyboard shortcuts
document.addEventListener('keydown', (e) => {
  if (e.ctrlKey && e.key === 'z') {
    e.preventDefault();
    const state = history.undo(currentSceneState);
    if (state) currentSceneState = state;
  }
  if (e.ctrlKey && e.key === 'y') {
    e.preventDefault();
    const state = history.redo(currentSceneState);
    if (state) currentSceneState = state;
  }
});
What's happening: The SceneHistory class manages two stacks. Before each modification, the current state is saved to undoStack. Ctrl+Z pops the last state and rebuilds the scene. Ctrl+Y restores forward. A max history of 20 states prevents memory bloat in long sessions.

  2. Keywords for additive: "add," "put," "place," "create," "insert"
  3. Keywords for transformative: "change," "make," "rotate," "move," "resize," "color"
  4. Keywords for subtractive: "remove," "delete," "clear," "get rid of"
  5. Display the detected command type as a badge in the UI before sending to the API
  6. Note: this is for UI feedback only — the AI still determines the actual action
Classification hint 
function classifyCommand(text) {
  const t = text.toLowerCase();
  if (/\b(add|put|place|create|insert)\b/.test(t)) return 'additive';
  if (/\b(remove|delete|clear|get rid)\b/.test(t)) return 'subtractive';
  return 'transformative'; // default
}
