Day 1: Introduction to 3D on the Web & Three.js

🟢 Beginner · ~1 hour

🎯 Learning Objectives

  • Describe the difference between WebGL and Three.js
  • Identify real-world applications of 3D on the web
  • Set up a basic development environment for Three.js

📚 Core Concepts

What is Three.js?

Think of Three.js like LEGO blocks for 3D web graphics. Just as LEGO provides pre-made bricks that snap together to build complex structures, Three.js provides pre-built components that work together to create 3D scenes in your browser. Without it, you'd be writing thousands of lines of low-level WebGL code just to draw a simple cube.

Three.js is a JavaScript library that makes WebGL (Web Graphics Library) accessible and manageable. WebGL is a powerful but complex API that lets browsers render 3D graphics using the GPU. While WebGL gives you direct control over the graphics pipeline, Three.js wraps that complexity in an intuitive API. Instead of writing shader code and managing buffer arrays, you can create 3D objects, apply materials, add lighting, and animate scenes with just a few lines of JavaScript.

The library was created by Ricardo Cabello (Mr.doob) in 2010 and has become the industry standard for 3D web graphics. It's used by companies like Google, NASA, Apple, and thousands of creative studios worldwide.

Why 3D on the Web Matters

Accessibility: No downloads, no installations. Users can access 3D experiences instantly through their browser. A VR museum tour, a product configurator, or an interactive data visualization is just a URL away.

Reach: With WebGL support in all modern browsers, your 3D content can reach billions of devices — from desktop computers to smartphones and VR headsets.

Integration: 3D can seamlessly integrate with HTML, CSS, and JavaScript. You can overlay UI elements on 3D scenes, respond to DOM events, and use the entire web platform ecosystem.

The Three Essential Components

Every Three.js application revolves around three core concepts (hence the name "Three.js"):

Scene I'm the 3D world — a container that holds all objects, lights, and cameras. Think of me like a movie set.
Camera I define what the viewer sees. I can be a PerspectiveCamera (realistic depth) or OrthographicCamera (no distortion).
Renderer I take the Scene and Camera and draw them onto an HTML canvas. I handle all the GPU communication.

These three work together in a continuous loop: the Renderer repeatedly draws the Scene as viewed through the Camera, creating smooth animation at 60 FPS.

💡 Key Insight: Three.js democratizes 3D web development by handling the complex WebGL code, letting you focus on creativity rather than low-level graphics programming.

WebGL vs Three.js

To draw a single colored triangle in raw WebGL, you need roughly 100+ lines of code — vertex shaders, fragment shaders, buffer creation, attribute binding, and more. In Three.js, you can create a full 3D scene with lighting and animation in about 20 lines. Three.js doesn't replace WebGL; it sits on top of it, translating your high-level instructions into the low-level commands the GPU understands.

Real-World Applications

  • E-commerce: Product configurators (Nike shoe customizer, furniture visualizers)
  • Education: Interactive 3D models (molecular structures, historical reconstructions)
  • Data Visualization: Complex datasets rendered in 3D space
  • Gaming: Browser-based games and interactive experiences
  • Portfolios: Immersive developer portfolios that stand out from flat websites

🔑 Key Terminology

WebGL
Low-level JavaScript API for rendering 3D graphics using the GPU. The foundation that Three.js is built upon.
Scene
Container that holds all 3D objects, lights, and cameras. const scene = new THREE.Scene()
Camera
Defines the viewpoint from which the scene is rendered. PerspectiveCamera for realistic depth.
Renderer
Takes the scene and camera and renders them to a canvas element. WebGLRenderer converts your 3D world into pixels.
Mesh
Combination of geometry (shape) and material (appearance). A cube with a red surface is a mesh.
Geometry
The shape/structure of a 3D object — vertices and faces. BoxGeometry, SphereGeometry, custom shapes.
Material
Defines how a surface looks — color, texture, shininess. MeshBasicMaterial, MeshStandardMaterial.
Canvas
HTML element where 3D graphics are drawn. Three.js creates and manages this for you.

💻 Code Examples

Example 1: Basic Three.js HTML Template

index.html Working Example 
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>My First Three.js Scene</title>
    <style>
        body { margin: 0; overflow: hidden; }
        canvas { display: block; }
    </style>
</head>
<body>
    <script type="module" src="/main.js"></script>
</body>
</html>
What's happening: This minimal HTML sets up a full-screen 3D canvas. CSS removes margins and scrollbars. The script loads as a module for ES6 imports.

Example 2: Installing Three.js with Vite

terminal 
mkdir threejs-ai-portfolio
cd threejs-ai-portfolio
npm create vite@latest . -- --template vanilla
npm install three
npm run dev
What's happening: Vite provides instant server start and hot module replacement — the recommended setup for Three.js projects.

Example 3: The Render Loop Pattern

main.js Conceptual 
// Setup (runs once)
const scene = new THREE.Scene();
const camera = new THREE.PerspectiveCamera(75, window.innerWidth / window.innerHeight);
const renderer = new THREE.WebGLRenderer();

// Animation loop (runs 60 times per second)
function animate() {
    requestAnimationFrame(animate);  // Schedule next frame
    // Update objects here (rotate, move, etc.)
    renderer.render(scene, camera);  // Draw the frame
}
animate();
What's happening: This is the heartbeat of Three.js. requestAnimationFrame calls your function before each screen repaint (~60 FPS). Update → render → repeat.
