Day 2: Your First Three.js Scene — The Rotating Cube

🟢 Beginner · ~1 hour

🎯 Learning Objectives

  • Create a complete Three.js scene with all required components
  • Add a 3D mesh (geometry + material) to your scene
  • Implement a render loop for animation
  • Position and rotate objects in 3D space
  • Handle window resizing for responsive 3D graphics

📚 Core Concepts

The Scene Graph

Think of a Three.js scene like a family tree. The Scene object is the root, and everything else — cameras, lights, meshes — are children. When you add an object with scene.add(mesh), you're creating a parent-child relationship. This hierarchy matters because transformations (position, rotation, scale) apply to children relative to their parents.

This parent-child system is called the "scene graph." If you rotate a parent object, all its children rotate with it. Imagine a solar system where planets orbit the sun, and moons orbit planets. The scene graph handles all that nested motion automatically.

Meshes: Geometry Meets Material

A Mesh is the most fundamental 3D object in Three.js. It combines two concepts:

Geometry — The shape. A collection of vertices (points in 3D space) and faces (triangles connecting those points). Three.js provides built-in geometries like BoxGeometry, SphereGeometry, and PlaneGeometry.

Material — The appearance. How the surface reflects light, its color, texture, and visual properties. It's like 3D printing: the geometry is the model file, and the material is whether you print in red plastic, metal, or wood.

The Coordinate System

X-axis: Left (−) → Right (+) Y-axis: Down (−) → Up (+) Z-axis: Back (−) → Front (+) toward camera

Origin (0, 0, 0) is the center. Position (2, 1, -5) = 2 right, 1 up, 5 away from camera.

The Render Loop

Animation in Three.js happens through continuous re-rendering. requestAnimationFrame tells the browser to call your function before the next screen refresh — typically 60 times per second.

1️⃣
Update State
Rotate, move, change
2️⃣
Render Scene
Draw the frame
3️⃣
Request Next
Schedule callback
🔄
Repeat
60 FPS forever
💡 Key Insight: Three.js is declarative — you describe what you want (a red cube at position X), not how to draw it. The renderer figures out the WebGL commands needed.

🔑 Key Terminology

Mesh
Combination of geometry (shape) and material (appearance). new THREE.Mesh(geometry, material)
BoxGeometry
Built-in geometry for boxes/cubes. new THREE.BoxGeometry(1, 1, 1) = 1×1×1 cube.
MeshBasicMaterial
Simplest material — flat color, no lighting effects. Good for debugging.
PerspectiveCamera
Camera mimicking human eye perspective. Parameters: FOV, aspect ratio, near, far.
requestAnimationFrame
Browser API for scheduling smooth animations. Syncs with screen refresh (~60 FPS).
Scene Graph
Tree structure of parent-child relationships. Children inherit parent transforms.
Aspect Ratio
Width ÷ height (e.g., 16:9 = 1.777). Prevents distortion in camera view.

💻 Code Examples

Example 1: Complete Rotating Cube

main.js Working Example 
import * as THREE from 'three';

// 1. Create the scene
const scene = new THREE.Scene();

// 2. Create the camera
const camera = new THREE.PerspectiveCamera(
    75,                                      // FOV in degrees
    window.innerWidth / window.innerHeight,  // Aspect ratio
    0.1,                                     // Near clip
    1000                                     // Far clip
);
camera.position.z = 5;  // Move camera back

// 3. Create the renderer
const renderer = new THREE.WebGLRenderer();
renderer.setSize(window.innerWidth, window.innerHeight);
document.body.appendChild(renderer.domElement);

// 4. Create a cube
const geometry = new THREE.BoxGeometry(1, 1, 1);
const material = new THREE.MeshBasicMaterial({ color: 0x00ff00 });
const cube = new THREE.Mesh(geometry, material);
scene.add(cube);

// 5. Animation loop
function animate() {
    requestAnimationFrame(animate);
    cube.rotation.x += 0.01;
    cube.rotation.y += 0.01;
    renderer.render(scene, camera);
}
animate();
What's happening: This is the "Hello World" of Three.js. We create the big three (scene, camera, renderer), add a green cube, and start an animation loop that rotates it slightly each frame.

Example 2: Handling Window Resize

resize-handler.js 
window.addEventListener('resize', onWindowResize);

function onWindowResize() {
    camera.aspect = window.innerWidth / window.innerHeight;
    camera.updateProjectionMatrix();  // MUST call this!
    renderer.setSize(window.innerWidth, window.innerHeight);
}
What's happening: When the window resizes, update the camera aspect ratio and renderer size. updateProjectionMatrix() is crucial — it recalculates the camera's internal math.

Example 3: Material Variations

materials.js 
// Hex colors
const red = new THREE.MeshBasicMaterial({ color: 0xff0000 });
const blue = new THREE.MeshBasicMaterial({ color: 0x0000ff });

// CSS color names
const yellow = new THREE.MeshBasicMaterial({ color: 'yellow' });

// Transparent
const ghost = new THREE.MeshBasicMaterial({
    color: 0x00ff00, transparent: true, opacity: 0.5
});

// Wireframe (great for debugging)
const wire = new THREE.MeshBasicMaterial({
    color: 0xffffff, wireframe: true
});
What's happening: Materials go beyond color — transparency, opacity, and wireframe mode reveal different aspects of your objects. Hex (0xRRGGBB) or CSS color names both work.

✏️ Hands-On Exercises

Exercise 1: Create Your First Scene

⏱️ ~20 min

Goal: Build a working Three.js scene from scratch

  1. Open your project from Day 1
  2. Replace main.js with Example 1 above
  3. Run npm run dev and open the browser
  4. Verify you see a green rotating cube
  5. Open the browser console — check for errors
🚀 Stretch: Add a second cube at a different position with a different rotation speed.

Exercise 2: Experiment with Colors & Rotation

⏱️ ~15 min

Goal: Understand how to modify mesh properties

  1. Change the cube's color to your favorite color
  2. Try transparent (opacity 0.5)
  3. Change rotation speeds (0.001, 0.05, negative values)
  4. Rotate around only one axis
  5. Try wireframe mode
Need a hint?

Colors: hex (0xFF5733) or names ('orange'). Rotation values are radians per frame — smaller = slower.

Exercise 3: Multi-Cube Scene

⏱️ ~25 min

Goal: Create a scene with multiple objects at different positions

  1. Create 3 cubes with different colors
  2. Position at (-2, 0, 0), (0, 0, 0), and (2, 0, 0)
  3. Different rotation speeds for each
  4. Add a wireframe cube at (0, 2, 0) — no rotation
  5. Add window resize handling
Need a hint?

Use mesh.position.set(x, y, z) after creating each cube. Give each a variable name to reference in the animate function.
