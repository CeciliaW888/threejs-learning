📚 Core Concepts

Material Types: From Simple to Realistic

If geometry is the skeleton of a 3D object, materials are its skin. They determine how surfaces look — whether they're shiny or matte, transparent or opaque, and how they interact with light in the scene.

💡
The golden rule: Use the simplest material that achieves your visual goal. MeshBasicMaterial is roughly 10x faster than MeshStandardMaterial because it skips all lighting calculations.

MeshBasicMaterial — Flat color, ignores lighting entirely. Like construction paper: flat, uniform, no depth. Perfect for debugging, wireframes, or flat-shaded aesthetics.

MeshLambertMaterial — Responds to light with diffuse (matte) reflection. Looks like chalk, unpolished wood, or fabric. No shiny spots.

MeshPhongMaterial — Adds specular highlights (shiny spots). Good for plastics, polished surfaces. The shininess property controls how focused the highlights are.

MeshStandardMaterial — Physically-Based Rendering (PBR), the industry standard. Uses intuitive roughness (0=mirror, 1=matte) and metalness (0=plastic, 1=gold) properties.

MeshPhysicalMaterial — Extends Standard with clearcoat (car paint), sheen (fabric), and transmission (glass). Most realistic but most expensive.

Textures: Wrapping Images on Geometry

A texture is an image mapped onto a 3D surface using UV mapping. Instead of a flat red cube, you can wrap a brick photograph onto it for a realistic look.

You can layer multiple texture types on one material:

  • map — Base color (albedo/diffuse)
  • normalMap — Fakes surface bumps using RGB data
  • roughnessMap — Varies roughness across the surface
  • metalnessMap — Varies metallic properties
  • aoMap — Ambient occlusion (darkens crevices)

Loading Textures

Three.js loads textures asynchronously with TextureLoader. You can use the texture immediately — Three.js renders blank until the image loads, then updates automatically.

Texture Properties

Once loaded, textures have properties controlling how they're applied:

  • repeat — How many times to tile (e.g., 4×4 grid)
  • wrapS / wrapT — Edge behavior: RepeatWrapping, MirroredRepeatWrapping, ClampToEdgeWrapping
  • rotation — Rotate the texture (radians)
  • offset — Shift position (animate for scrolling effects!)
🎯
Key Insight: Materials + Textures = Visual realism. Geometry defines shape, materials define light interaction, textures add visual detail. Together, they transform simple shapes into convincing objects.

🔑 Key Terminology

Click each card to reveal the definition. Use arrow keys to navigate.

Diffuse
Light scattered equally in all directions — creates a matte appearance. Examples: chalk, unpolished wood, fabric.
Specular
Light reflected in a mirror-like direction — creates shiny spots. Examples: plastic, polished metal, water surfaces.
PBR (Physically-Based Rendering)
Rendering approach that simulates realistic light behavior using roughness and metalness properties. Used in MeshStandardMaterial — the industry standard for 3D.
UV Mapping
2D coordinates (0-1 range) that map texture pixels to geometry vertices. Like unwrapping a 3D box into a flat pattern for painting.
Albedo
The base color of a surface without any lighting or shadows applied — the "true" color you'd see in perfectly neutral light.
Normal Map
An RGB image encoding surface normals to fake bumps and detail without adding geometry. Makes a flat wall look like rough bricks.
Metalness
How metallic a surface appears. 0 = non-metal (rubber, plastic), 0.5 = brushed steel, 1.0 = fully metallic (gold, chrome).
Roughness
How rough or smooth a surface is. 0 = perfect mirror, 0.5 = satin finish, 1.0 = sandpaper. Controls how focused reflections are.

💻 Code Examples

Example 1: Material Comparison

material-comparison.js Three.js 
// Four identical spheres with different materials
const geometry = new THREE.SphereGeometry(1, 32, 32);

const materials = [
    new THREE.MeshBasicMaterial({ color: 0xff0000 }),
    new THREE.MeshLambertMaterial({ color: 0xff0000 }),
    new THREE.MeshPhongMaterial({ color: 0xff0000, shininess: 100 }),
    new THREE.MeshStandardMaterial({
        color: 0xff0000,
        roughness: 0.5,
        metalness: 0
    })
];

materials.forEach((mat, i) => {
    const mesh = new THREE.Mesh(geometry, mat);
    mesh.position.x = (i - 1.5) * 2.5;
    scene.add(mesh);
});
What's happening: Four identical spheres, each with a different material, reveal how material choice affects appearance. BasicMaterial ignores light. Lambert shows soft gradients. Phong adds specular highlights. Standard uses PBR for the most realistic look.

Example 2: Loading & Applying Textures

textured-cube.js Three.js 
// Load and apply a UV test texture
const textureLoader = new THREE.TextureLoader();
const texture = textureLoader.load(
    'https://threejs.org/examples/textures/uv_grid_opengl.jpg',
    () => { console.log('Texture loaded!'); },
    undefined,
    (err) => { console.error('Error:', err); }
);

// Apply texture as the base color map
const geometry = new THREE.BoxGeometry(2, 2, 2);
const material = new THREE.MeshBasicMaterial({ map: texture });
const cube = new THREE.Mesh(geometry, material);
What's happening: TextureLoader fetches an image asynchronously and applies it as the map property. The UV grid texture shows exactly how the image wraps around each face of the cube.

Example 3: Texture Transformation

texture-transforms.js Three.js 
// Repeating texture — tiles 4×4 across the surface
const tiledTexture = textureLoader.load('brick.jpg');
tiledTexture.wrapS = THREE.RepeatWrapping;
tiledTexture.wrapT = THREE.RepeatWrapping;
tiledTexture.repeat.set(4, 4);

// Rotated texture — spin 45° around center
const rotatedTexture = textureLoader.load('wood.jpg');
rotatedTexture.rotation = Math.PI / 4;
rotatedTexture.center.set(0.5, 0.5);

// Animate offset for scrolling water effect
// In render loop: scrollTexture.offset.x += 0.001;
What's happening: Texture properties control mapping. repeat creates tiling (great for floors/walls). rotation spins the texture. Animating offset creates scrolling effects like flowing water.
💡 Try it: Load a texture and animate its offset in the render loop for a scrolling water effect. Then try animating rotation for a spinning vortex!
