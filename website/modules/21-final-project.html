Project Requirements

What We're Building

An AI-powered 3D portfolio where:

  1. Landing Scene — User arrives at a personalized 3D workspace that represents YOU
  2. Conversational Interface — Chat with AI to explore your portfolio
  3. Project Showcases — 3D representations of your projects (e.g., "show me your React work")
  4. Dynamic Scenes — AI generates different scenes based on what the user wants to see
  5. About/Contact — AI-driven storytelling of your background and skills

Core Features Checklist

3D
Scene Generation (Week 1-2)

Three.js scene with lighting, materials, shadows. Load 3D models (optional: your logo, custom objects). Performance optimized (60fps, <500 draw calls). Responsive (works on desktop + mobile).

AI
AI Integration (Week 3)

Claude/OpenAI API integration via serverless function. Conversational scene modification. Memory of user preferences and conversation history. Chain-of-thought scene generation.

Content
Portfolio Content

User profile with skills, projects, bio. Project database (3-5 real projects). AI generates scenes representing each project. Contact information accessible.

Polish
Polish

Loading states, error handling. Smooth scene transitions. UI/UX: clean input, readable text. Analytics (optional: track popular scenes).

1. Landing Experience

First visit: A 3D workspace scene fades in. The AI greets the user: "Hi! I'm an AI-powered portfolio. Ask me about your projects, skills, or tell me what kind of scene you'd like to see."

Users can type natural language requests, click pre-made prompts ("Show me React projects", "Tell me about you"), or explore the 3D scene with the mouse.

2. Intent Classification

intent-classifier.js 
function classifyIntent(userInput) {
    const input = userInput.toLowerCase();

    if (input.includes('project') || input.includes('work')) {
        return { intent: 'show-projects', extract: extractProjectKeywords(input) };
    }
    if (input.includes('about') || input.includes('who')) {
        return { intent: 'show-bio' };
    }
    if (input.includes('contact') || input.includes('email')) {
        return { intent: 'show-contact' };
    }
    if (input.includes('skill') || input.includes('tech')) {
        return { intent: 'show-skills' };
    }
    if (input.includes('add') || input.includes('change') || input.includes('remove')) {
        return { intent: 'modify-scene', modification: input };
    }

    return { intent: 'generate-custom', prompt: input };
}

4. Dynamic Scene Router

scene-router.js 
async function handleUserInput(input) {
    const { intent, ...data } = classifyIntent(input);

    switch (intent) {
        case 'show-projects':
            const projects = filterProjects(data.extract);
            await showProjectShowcase(projects);
            break;
        case 'show-bio':
            await showBioScene();
            break;
        case 'show-contact':
            await showContactScene();
            break;
        case 'modify-scene':
            await modifyCurrentScene(data.modification);
            break;
        case 'generate-custom':
            await generateCustomScene(data.prompt);
            break;
    }
}

Final Project Tasks

Phase 1: Setup & Structure

~30 min

Goal: Scaffold the project

  1. Create project repository
  2. Set up Vite + Three.js + AI integration
  3. Create folder structure: /src/components, /src/ai, /src/scenes, /api
  4. Configure environment variables
  5. Deploy initial version to Vercel

Phase 2: Portfolio Content

~45 min

Goal: Define your portfolio data and scene templates

  1. Define your portfolioData object (bio, skills, 3-5 projects)
  2. Create user profile for AI context
  3. Design 3-5 pre-made scene templates for projects
  4. Write welcome message and sample prompts

Phase 3: AI Integration

~60 min

Goal: Connect all AI features

  1. Set up serverless API endpoint
  2. Implement intent classification
  3. Connect Claude API with conversation memory
  4. Add project-specific scene generation
  5. Test all intent paths

Phase 4: Polish & Testing

~45 min

Goal: Production-quality user experience

  1. Add loading states and error handling
  2. Implement scene transitions
  3. Responsive design (test on mobile)
  4. Performance audit (60fps target)
  5. Test all features end-to-end

~20 min

Goal: Ship it!

  2. Test on multiple devices/browsers
