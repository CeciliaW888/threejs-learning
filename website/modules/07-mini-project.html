The Minimum Viable 3D Scene

Every Three.js scene needs these 8 pieces working together: Scene (container), Camera (viewpoint), Renderer (draws to canvas), Geometry (shape), Material (appearance), Mesh (geometry + material), Light (illumination), and an Animation Loop (continuous rendering). If any piece is missing, you'll see a black screen.

Common Gotchas

⚠️
Black Screen Debugging Checklist
  • Camera at (0,0,0) — you're inside objects. Move it back!
  • StandardMaterial with no lights — add ambient + directional
  • Camera looking wrong way — try camera.lookAt(0, 0, 0)
  • Renderer not in DOM — check document.body.appendChild(renderer.domElement)
  • Forgot updateProjectionMatrix() after camera changes
  • Forgot controls.update() in loop with damping
💡
Key Insight

Most Three.js problems come from missing configuration, not code errors. When something doesn't look right, check your setup chain before debugging complex logic.

🔑 Key Terminology Review

Scene Graph
Tree of parent-child 3D objects. Groups can contain meshes, lights, and other groups. (Day 2-3)
PBR
Physically-Based Rendering — realistic material model using roughness and metalness to simulate real-world surfaces. (Day 4)
Three-Point Lighting
Professional lighting setup: Key light (main), Fill light (softens shadows), Back light (separates subject from background). (Day 5)
Shadow Map
A depth texture rendered from the light's perspective. Used to determine which areas are in shadow. (Day 5)
OrbitControls
Mouse/touch camera navigation — left-click orbits, right-click pans, scroll zooms. Standard for 3D viewers. (Day 6)
Mesh
Geometry + Material = visible object. The fundamental building block of any Three.js scene. (Day 2)
FOV
Field of View — the camera's vertical viewing angle in degrees. 75° is a good default. (Day 6)
Texture Mapping
Wrapping 2D images onto 3D surfaces. Loaded with TextureLoader and applied to material properties. (Day 4)

💻 Mini-Project: Interactive 3D Gallery

Build a 3D art gallery with multiple objects on a floor, professional lighting, shadows, and orbit controls. This combines everything from Week 1.

gallery-starter.js Starter Code 
import * as THREE from 'three';
import { OrbitControls } from 'three/examples/jsm/controls/OrbitControls.js';

// ===== SCENE SETUP =====
const scene = new THREE.Scene();
scene.background = new THREE.Color(0x1a1a2e);

const camera = new THREE.PerspectiveCamera(
    75, window.innerWidth / window.innerHeight, 0.1, 1000
);
camera.position.set(8, 5, 8);

const renderer = new THREE.WebGLRenderer({ antialias: true });
renderer.setSize(window.innerWidth, window.innerHeight);
renderer.shadowMap.enabled = true;
renderer.shadowMap.type = THREE.PCFSoftShadowMap;
document.body.appendChild(renderer.domElement);

const controls = new OrbitControls(camera, renderer.domElement);
controls.enableDamping = true;
controls.dampingFactor = 0.05;

// ===== FLOOR =====
const floor = new THREE.Mesh(
    new THREE.PlaneGeometry(20, 20),
    new THREE.MeshStandardMaterial({ color: 0x404040, roughness: 0.8 })
);
floor.rotation.x = -Math.PI / 2;
floor.receiveShadow = true;
scene.add(floor);

// ===== GALLERY OBJECTS =====
// Object 1: Metallic red sphere
const sphere = new THREE.Mesh(
    new THREE.SphereGeometry(1, 32, 32),
    new THREE.MeshStandardMaterial({
        color: 0xff0000, roughness: 0.3, metalness: 0.7
    })
);
sphere.position.set(-3, 1, 0);
sphere.castShadow = true;
scene.add(sphere);

✏️ Hands-On Exercises

Exercise 1: Quick Knowledge Check

⏱️ ~10 min

Goal: Test your Week 1 retention

  1. Answer these 5 questions without looking at notes
  2. Then check your answers against the course material

Q1: What three objects does every Three.js scene require at minimum?

✅ Correct! Scene (container), Camera (viewpoint), and Renderer (draws to canvas) are the essential trio.

❌ The essential trio is Scene, Camera, and Renderer. Without all three, nothing renders.

Q2: What's the key difference between MeshBasicMaterial and MeshStandardMaterial?

✅ Correct! MeshBasicMaterial is always fully lit (ignores lights). MeshStandardMaterial uses physically-based rendering and reacts realistically to lights.

❌ The key difference: BasicMaterial ignores all lights (always fully lit), while StandardMaterial uses PBR and reacts to lights realistically.

Q3: What are the 3 steps to enable shadows?

✅ Correct! Three steps: 1) renderer.shadowMap.enabled = true, 2) light.castShadow = true, 3) objects: castShadow = true and/or receiveShadow = true.

❌ Three steps needed: enable on the renderer, enable on the light source, and configure each mesh to cast and/or receive shadows.

Q4: What does FOV control in PerspectiveCamera?

✅ Correct! FOV is the vertical viewing angle in degrees. 75° is a good default; wider = more visible, narrower = zoomed in.

❌ FOV (Field of View) is the vertical viewing angle in degrees. Higher values show more of the scene (wide-angle), lower values show less (telephoto/zoom).

Q5: Why must you call updateProjectionMatrix() after changing camera properties?

✅ Correct! The camera caches its projection matrix for performance. When you change FOV, aspect, near, or far, you must tell it to recalculate.

❌ The camera uses an internal projection matrix for rendering. When properties change, this matrix must be recalculated manually — Three.js doesn't do it automatically for performance reasons.

Lighting hint

Start with AmbientLight(0xffffff, 0.3) for base, then DirectionalLight(0xffffff, 1.0) at position (5, 10, 5) as key light with castShadow = true. Add a dimmer point light from the other side as fill.

Layout hint

Arrange objects in a semicircle or grid. Use different Y positions for visual interest. Try grouping a cylinder pedestal + object on top using THREE.Group().
