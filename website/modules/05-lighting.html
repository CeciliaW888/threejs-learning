📚 Core Concepts

Why Lighting Matters

Lighting is what makes 3D scenes feel real. Without lights, only MeshBasicMaterial is visible — everything else renders pitch black. The same model can look friendly or ominous based purely on how you light it.

Think of it like photography. A portrait photographer positions key lights, fill lights, and backlights to shape a subject. Three.js gives you the same tools.

The Four Essential Light Types

🌍 AmbientLight: I light everything equally from all directions. No shadows, no drama. I'm the baseline — keeping things from going pitch black.
☀️ DirectionalLight: I send parallel rays, like the sun. My position sets my direction, but distance doesn't matter. I'm perfect for outdoor scenes.
💡 PointLight: I radiate from a single point in all directions — like a bare light bulb. My brightness fades with distance. Use me for lamps and torches!
🔦 SpotLight: I'm the most dramatic — a focused cone of light. I have angle, penumbra (soft edges), and a target. Stage lighting? Car headlights? That's me.

Light Properties

All lights share: color (hex value), intensity (brightness multiplier), position (3D coordinates), and castShadow (expensive, off by default).

Additional properties by type:

  • PointLight / SpotLight: distance (range), decay (falloff rate, 2 = realistic)
  • SpotLight: angle (cone width), penumbra (soft edge, 0-1)

Shadows: Adding Depth

Shadows ground objects in a scene — without them, everything looks like it's floating. But they're expensive because the GPU must render the scene again from the light's perspective.

⚠️
Three steps to enable shadows:
1. renderer.shadowMap.enabled = true
2. light.castShadow = true
3. mesh.castShadow = true and ground.receiveShadow = true
Miss any one of these and shadows won't appear!

Professional Lighting Setups

Three-Point Lighting is the industry standard:

  • Key light — Main light, brightest, creates primary shadows
  • Fill light — Softer, opposite side, reduces harsh shadows
  • Back light — Behind subject, creates rim/edge highlights
🎯
Key Insight: Lighting is as important as modeling. Great lighting can make simple geometry look amazing; poor lighting can ruin detailed models. Spend as much time on lights as you do on objects.

🔑 Key Terminology

Click each card to reveal the definition. Use arrow keys to navigate.

AmbientLight
Uniform lighting from all directions — no shadows, no directionality. Acts as baseline illumination to prevent pitch-black areas.
DirectionalLight
Parallel rays like sunlight. Position defines direction, but distance doesn't matter. Creates consistent shadows across the entire scene.
PointLight
Radiates from a single point in all directions, like a light bulb. Intensity decreases with distance. Great for lamps and torches.
SpotLight
Focused cone of light like a flashlight. Has angle (cone width) and penumbra (soft edge). Most dramatic and cinematic light type.
Intensity
Brightness multiplier for lights. 1 = normal, 2 = double bright, 0.5 = half. Controls how strong the light appears.
Shadow Map
A depth texture rendered from the light's perspective. Higher resolution (e.g., 2048) creates sharper shadows but uses more GPU memory.
Penumbra
The soft edge on a spotlight cone. 0 = hard edge, 1 = very soft falloff. Creates realistic, natural-looking light boundaries.
Light Decay
How quickly light fades with distance. decay=2 is physically accurate (inverse-square law). Only applies to PointLight and SpotLight.

💻 Code Examples

Example 1: Basic Lighting Setup

basic-lighting.js Three.js 
// A sphere on a plane with ambient + directional light
const sphere = new THREE.Mesh(
    new THREE.SphereGeometry(1, 32, 32),
    new THREE.MeshStandardMaterial({
        color: 0xff0000, roughness: 0.5
    })
);
sphere.position.y = 1;
scene.add(sphere);

const ground = new THREE.Mesh(
    new THREE.PlaneGeometry(10, 10),
    new THREE.MeshStandardMaterial({ color: 0x808080 })
);
ground.rotation.x = -Math.PI / 2; // Lay flat
scene.add(ground);

// Ambient prevents pitch-black shadows
const ambientLight = new THREE.AmbientLight(0x404040, 0.5);
scene.add(ambientLight);

// Directional creates highlights and depth
const dirLight = new THREE.DirectionalLight(0xffffff, 1);
dirLight.position.set(5, 10, 5);
scene.add(dirLight);
What's happening: The ambient light provides base illumination — without it, the sphere's shadow side would be completely black. The directional light creates the bright-to-dark gradient that our brains interpret as 3D depth.

Example 2: Enabling Shadows

shadows.js Three.js 
// Step 1: Enable on renderer
renderer.shadowMap.enabled = true;
renderer.shadowMap.type = THREE.PCFSoftShadowMap;

// Step 2: Tell light to cast shadows
dirLight.castShadow = true;
dirLight.shadow.camera.left = -10;
dirLight.shadow.camera.right = 10;
dirLight.shadow.camera.top = 10;
dirLight.shadow.camera.bottom = -10;
dirLight.shadow.mapSize.width = 2048;
dirLight.shadow.mapSize.height = 2048;

// Step 3: Objects opt in
sphere.castShadow = true;
ground.receiveShadow = true;

// Debug: visualize shadow camera
const helper = new THREE.CameraHelper(dirLight.shadow.camera);
scene.add(helper);
What's happening: Shadows require the GPU to render from the light's perspective, creating a depth map. The shadow camera frustum defines the shadow area. Higher mapSize = sharper shadows. PCFSoftShadowMap creates soft, realistic edges.

Example 3: Three-Point Lighting

three-point-lighting.js Three.js 
// Key light — main, bright, from front-right
const keyLight = new THREE.DirectionalLight(0xffffff, 1);
keyLight.position.set(5, 5, 5);
keyLight.castShadow = true;
scene.add(keyLight);

// Fill light — softer, from left, slightly blue
const fillLight = new THREE.DirectionalLight(0x8080ff, 0.4);
fillLight.position.set(-5, 3, 0);
scene.add(fillLight);

// Back light — warm rim behind subject
const backLight = new THREE.DirectionalLight(0xffff80, 0.6);
backLight.position.set(0, 3, -5);
scene.add(backLight);

// Subtle ambient fill
const ambient = new THREE.AmbientLight(0x404040, 0.2);
scene.add(ambient);
What's happening: Professional three-point setup. Key light creates primary shadows. Blue-tinted fill brightens the shadow side. Warm back light creates a rim around the subject, separating it from the background. Together they give objects a 3D, cinematic quality.
💡 Try it: Change the key light to red and the fill to blue for a dramatic neon look. Then move the key light directly above — how does the mood change?

⏱️ ~25 min 🟡 Intermediate

Goal: Enable and configure realistic shadows

1
Create objects on a ground plane

Sphere, cube, and cylinder above a PlaneGeometry

2
Enable all three shadow steps

Renderer → Light → Objects (castShadow + receiveShadow)

3
Debug with CameraHelper

Visualize the shadow camera frustum and adjust to cover all objects

💡 Hint: Shadows not appearing?

Check all four requirements: renderer.shadowMap.enabled, light.castShadow, object.castShadow, ground.receiveShadow. All four must be true!
